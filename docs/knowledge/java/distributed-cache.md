# 分布式缓存高频

## 本地缓存 vs 分布式缓存

:::info 知识点
缓存分类、Guava/Caffeine、Redis/Memcached
:::

* **本地缓存**：数据存在 JVM 内存中。读取极快，但无法多节点共享，适合存生命周期短、数据量小且不常变动的数据（如 Mybatis 一二级缓存）；
* **分布式缓存**：独立部署的服务（如 Redis）。支持多节点共享、高可用、持久化，适合大规模分布式系统。

## Redis 常见数据结构及应用场景

:::info 知识点
String、List、Set、ZSet、Hash、Bitmap、GEO
:::

* **String**：计数器、分布式锁、分布式 Session；
* **List**：消息队列（lpush/rpop）、最新联系人列表；
* **Hash**：存储对象信息（对比 String 节省空间）、购物车；
* **Set**：去重、共同好友、抽奖小程序；
* **ZSet**：实时排行榜、优先级任务队列；
* **Bitmap**：日活/月活统计、布隆过滤器；
* **GEO**：附近的人/商家。

## Redis 为什么快？单线程还是多线程？

:::info 知识点
内存操作、IO 多路复用、线程模型
:::

* **核心原因**：全内存操作、高效数据结构、避免了上下文切换开销；
* **线程模型**：
    * **Redis < 6.0**：网络 IO 和命令执行均由单线程完成；
    * **Redis 6.0+**：**多线程处理网络 IO**，但**命令执行依然是单线程**（保证原子性和简单性）。

## 缓存雪崩、穿透、击穿及解决方案

:::info 知识点
缓存三难题、布隆过滤器、过期策略
:::

* **缓存雪崩**：大量 Key 同时过期。**解决**：过期时间加随机抖动、搭建高可用集群；
* **缓存穿透**：查询不存在的数据（绕过缓存打到 DB）。**解决**：接口参数校验、缓存空对象、**布隆过滤器**；
* **缓存击穿**：热点 Key 失效。**解决**：设置热点数据永不过期、加互斥锁（Mutex）。

## Redis 持久化机制：RDB vs AOF

:::info 知识点
快照、日志、混合持久化
:::

* **RDB (快照)**：周期性备份内存数据到磁盘。文件小、恢复快，但可能丢失最后一次快照后的数据；
* **AOF (日志)**：记录每个写命令。数据安全性高，但文件大、恢复慢；
* **混合持久化**：JDK 4.0 后支持。RDB 做全量，AOF 做增量，平衡性能与安全。

## Redis 淘汰策略与过期删除

:::info 知识点
定期删除、惰性删除、LRU/LFU 算法
:::

* **过期策略**：定期删除（随机抽查）+ 惰性删除（访问时检查）；
* **淘汰策略 (maxmemory-policy)**：内存满时触发。常用 `allkeys-lru`（最近最少使用）或 `allkeys-lfu`（使用频率最少）。

## 分布式锁的实现

:::info 知识点
原子性、Redlock
:::

* **原理**：使用 `setnx` (set if not exists) 抢占标识；
* **核心点**：必须设置过期时间（防止死锁）、Value 需具备唯一性（防止误删他人锁）、Lua 脚本保证操作原子性。
