# JVM 虚拟机高频

## JVM 运行时内存区域 (JDK 8+)

:::info 知识点
堆、方法区（元空间）、虚拟机栈、本地方法栈、程序计数器
:::

* **线程私有**：
    * **程序计数器**：记录当前线程执行的字节码行号；
    * **虚拟机栈**：存储局部变量表、操作数栈、方法出口等；
    * **本地方法栈**：为 Native 方法服务；
* **线程共享**：
    * **堆 (Heap)**：存放对象实例，GC 的主战场。分为新生代 (Eden, S0, S1) 和老年代；
    * **元空间 (Metaspace)**：逻辑上的方法区实现。存类信息、常量池。使用本地内存，不再受 JVM 堆限制。

## 垃圾回收算法及优缺点

:::info 知识点
标记清除、标记复制、标记整理、分代收集
:::

* **标记-清除**：简单，但会产生内存碎片；
* **标记-复制**：无碎片，速度快，但可用内存减半。适合新生代；
* **标记-整理**：解决碎片问题，但需移动对象，效率较低。适合老年代；
* **分代收集**：核心思想。新生代用复制算法，老年代用标记-清除或整理。

## 什么是 GC Root？可达性分析

:::info 知识点
可达性路径、引用链、对象存活判断
:::

* **GC Roots**：虚拟机栈引用、静态属性引用、常量引用、本地方法栈引用等；
* **判断依据**：从 GC Roots 开始向下搜索，搜索不到的对象即为不可用。

## 常见垃圾收集器及其组合

:::info 知识点
Serial、Parallel、CMS、G1、ZGC
:::

* **Serial/Serial Old**：单线程，STW 时间长，适合内存小的客户端；
* **Parallel Scavenge/Old**：吞吐量优先，适合后台运算；
* **CMS (Concurrent Mark Sweep)**：停顿时间优先。标记-清除算法，存在碎片；
* **G1 (Garbage First)**：JDK 9+ 默认。将堆分为多个 Region，局部整理，可预测停顿；
* **ZGC**：JDK 15+ 正式引入。低延迟（停顿 < 10ms），支持 TB 级内存。

## JVM 排查常用命令

:::info 知识点
jps、jinfo、jstat、jmap、jstack
:::

* `jps`：查看 Java 进程 ID；
* `jinfo`：查看/修改 JVM 参数；
* `jstat`：监控类加载、内存、GC 情况；
* `jmap`：生成堆转储快照 (Heap Dump)；
* `jstack`：查看线程堆栈（排查死锁、CPU 过高）。

## 类加载机制 (双亲委派)

:::info 知识点
加载器层级、委派模型、隔离与安全
:::

* **加载器层级**：Bootstrap (启动) -> Extension (扩展) -> App (应用) -> Custom (自定义)；
* **双亲委派**：收类加载请求后，先委托父类加载器尝试，父类无法加载时才自己加载；
* **作用**：防止核心 API 被篡改，保证类加载的唯一性和安全性。
