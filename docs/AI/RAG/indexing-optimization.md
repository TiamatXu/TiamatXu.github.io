# 索引优化 Indexing Optimization

传统 RAG 将文档切成固定大小的块并直接向量化，这会带来“信息瓶颈”和“上下文割裂”等问题。通过优化索引构建，可以显著提升检索精度。

## 多表示索引 (Multi-representation)

**核心思想**：用摘要检索，返回原文。

为每个文档块创建两份内容：
1. **摘要 (Summary)**：简洁、聚焦核心事实，用于向量匹配。
2. **原文 (Original Chunk)**：包含完整细节，检索命中后直接提供给 LLM。

::: details e.g. 财务长文档处理
- **痛点**：一份 50 页的财报，直接切块可能导致关键数据丢失上下文。
- **优化**：为每个章节生成一个包含“年份、公司名、核心数据项”的摘要。检索时通过摘要精准匹配，生成时使用详细的原文内容。
:::

## 递归树状检索 (RAPTOR)

**核心思想**：构建层次化的摘要树，解决跨层级的抽象问题。

1. 对原始文档块进行聚类。
2. 为每个集群生成一个“高层摘要”。
3. 对摘要再次聚类并生成更高层的摘要，直到形成一棵树。

**优势**：
- **低层次查询**：命中叶子节点（具体的财务报表项）。
- **高层次查询**：命中根节点或中间节点（如：“公司过去五年的整体财务稳健性评估”）。

## Token 级别检索 (ColBERT)

**核心思想**：细粒度交互。传统的向量检索是“一句话一个向量”，ColBERT 是“每个词一个向量”。

- **优点**：能够捕捉极其细微的语义匹配，对专有名词、缩写特别敏感。
- **代价**：存储开销巨大（通常是传统方法的 10 倍以上）。

## 对比与选型

| 方法 | 核心优势 | 适用场景 |
| :--- | :--- | :--- |
| **Multi-rep** | 简单有效，信噪比高 | 常规财务文档问答 |
| **RAPTOR** | 擅长总结全局信息 | 需要综合理解长篇报告的场景 |
| **ColBERT** | 极致的检索精度 | 法律、医疗等对术语匹配要求极高的领域 |
