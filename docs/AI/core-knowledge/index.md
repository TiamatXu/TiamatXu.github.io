# AI 相关核心知识

## LLM 和 Agent 的区别

**LLM (Large Language Model | 大语言模型)**  
本质上是一个**条件概率模型**，输入一个文本序列，输出下一个最可能的词，即输入一段 token，它预测下一个 token 最可能是什么。
`P(token_n | token_1, token_2, ..., token_{n-1})` 它通过大量的文本数据进行训练，学习语言的结构和语义关系。LLM
可以用于生成文本、回答问题、翻译等任务，但它没有自主决策能力。可以看作是一个**无状态函数**：输入 prompt，输出
response。它没有记忆和上下文感知能力，每次调用都是独立的，对外部世界一无所知。

LLM 的能力：

1. 只会根据输入的文本生成输出，无法主动获取信息或执行操作；能告诉你去哪查天气，但是无法自己去查天气
2. 没有记忆能力，无法保留之前的对话或上下文；上下文窗口满了就忘了之前的对话，跨会话完全失忆
3. 无法自主决策或规划，只能被动线性的响应输入
4. 训练数据有截止日期，近期发生的事模型不知道

**Agent (智能体)**  
Agent = LLM + 记忆 + 工具调用 + 规划 (本质是循环)  
它可以根据环境的反馈进行决策和行动。Agent 可以主动获取信息、执行操作、与外部系统交互，并且具有记忆能力，可以保留上下文和历史对话。Agent
可以自主规划和决策，适应动态环境。

Agent 四大模块：

1. **LLM (大脑)**：提供语言理解和生成能力，负责意图识别，推理判断
2. **规划模块**：任务拆解、步骤排序
3. **记忆模块**：长期记忆 (知识库)+ 短期记忆 (上下文窗口)
4. **工具调用模块**：外部 API 调用、数据库查询、代码执行、文件操作等

**本质区别：LLM 只能告诉你如何做，Agent 能直接帮你做**

## Agent 和 Workflow 的区别

**本质区别：流程由谁控制**

Workflow 是一个代码预定义的流程，按照固定的步骤执行，LLM 只是其中某些节点的处理器。  
Agent 的流程是由 LLM 动态生成的，接收到目标会自主规划执行路径，具有极强的灵活性。

|   维度    | Workflow | Agent  |
|:-------:|:--------:|:------:|
|   控制    |  开发者/代码  |  LLM   |
| Token消耗 |   低 1x   | 高 4-8x |
|   可预测   |    高     |   低    |
|   灵活性   |    低     |   高    |
|  使用场景   |   固定流程   | 开放式目标  |
|  调试难度   |    低     |   高    |

在实际使用中，两者混合使用才是正确的姿势，Workflow 负责核心流程控制，Agent 负责流程中的复杂决策和动态调整。

## Agent 工作模式

**ReAct (Reasoning + Acting)**  

推理 + 行动，最经典的 Agent 模式，几乎是所有主流框架 (LangChain、LangGraph) 的默认模式。  
**核心**：Thought (思考) → Action (行动) → Observation (观察) → Thought。根据当前状态进行思考，决定采取什么行动，执行后观察结果，再进行下一轮思考和行动。  
**优**：透明可审计 (每步思考可见)、灵活适应 (根据结果可动态调整)、通用性强  
**劣**：Token 消耗大 (每步都要完整推理)、死循环 (工具反复失败)、延迟高 (每次行动都要等 LLM 响应)  
**如何避免死循环**：最大步数限制 (通常超过 15 步就强制终止)、重复动作检测 (连续三次调用同一工具且参数相同，直接退出循环)、超时控制 (整个任务设置最大执行时间约束)

**Plan-and-Execute (先规划再执行)**

先计划清楚，再按计划逐步执行，不需要每步重复推理过程。Planner LLM 一次性输出完整的计划，Executor 按计划执行，每步只完成当前任务。可加入**重新规划检查点**，若计划不合理则重新规划执行，减少了 Thought 成本。  
Token 消耗：ReAct 每步都需要全局思考，假设消耗 100%，那么 Plan-and-Execute 只需规划一次 (理想状态)，执行相对更节省，消耗 20%。
